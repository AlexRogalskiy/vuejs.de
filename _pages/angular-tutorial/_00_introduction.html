Warum Angular?

Das Web war eine ziemlich lange Zeit einfach statisch war und bestand eigentlich nur aus Texten. Irgendwann wurden dann die ersten Anwendungen im Web geschrieben. Was damals dazu genutzt wurde war mehr oder weniger JavaScript oder Unterstützung eines Frameworks, was die Erstellung von Applikationen im Web einfacher macht. Angular hilft also dabei, Applikationen im Web um einiges einfacher zu entwickeln. Besonders, wenn es dabei um viel Interaktivität geht.

Konzepte und Grundideen von Angular

Bevor wir mit der ersten Aufgabe starten, erklären wir noch kurz, was Angular alles leisten kann und welche grundlegende Philosophie es verfolgt:

Philosophie

* Focus on maintainability * Prevents developers from doing the wrong thing * Tries staying close to web standards / dom

Was umfasst Angular alles.

Wie für viele andere Frameworks, gibt es natürlich eine Menge Erweiterungen von externen Entwicklern. Diese Teile stellen aber den Kern dar und werden von den Kern-Enwicklern geflegt: * Internationalisierung: .... * Animationen: Du hast eine Liste und möchtest * Server-Side-Rendering

Interoperabilität / Zukunft

Wir leben in der Zeit des Webs und der JavaScript-Bereich ändert sich sehr schnell. Deshalb ist die Frage schon berechtigt, wie Zunkunftssicher der Einsatz von Angular ist. ...

Performance

Um Performance-Aspekte auseinanderzunehmen, ist hier eigentlich die falsche Stelle. Doch ist dies oft eine erste Frage in Bezug auf Web-Frameworks. Deshalb hier nur kurz ein paar Buzz-Words, die wir in späteren Teilen noch genauer beschreiben. Angular unterstützt. * Pre-rendering (Server Side Rendering) * Offline compile (Ahead-of-Time compiling) * View caching * Web worker

Cross Plattform

Angular könnt ihr im Browser, auf dem Server und auf mobilen Plattformen einsetzen. Auf mobilen Plattformen gibt es dazu sogar 2 Möglichkeiten: * Ionic: Ihr benutzt Angular, wie ihr es im Web nutzt. Eure Anwendung wird als HTML-Seite in einem Web-Container gebaut. Durch Ionic bekommt ihr unterstützung für GUI-Elemente, die zur Plattform passen. * NativeScript: Ihr schreib Angular-Komponenten und diese werden in Komponenten für die Zielplattform umgewandelt. Schreibt ist zum Beispiel eine Liste, wird daraus eine native GUI-list für IOS.
