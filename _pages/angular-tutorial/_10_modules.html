Hier bietet sich eine Fachliche Aufteilung an. {% endcapture %} {% capture why %} [Liste daraus machen, um den Inhalt besser scannen zu können.] Eine größere Applikation braucht Struktur, um wartbar zu sein. Sie gruppieren normalerweise fachlich zusammenhängenden Code. Neben einer vernünftigen Verzeichnisstruktur, helfen Module auch beim Nachladen von Teilen der Applikation. Auch wenn man eine komplexe, große Applikation. Es ist sehr ungünstig, vom Benutzer zu verlangen, z.B. 8mb herunterzuladen, um überhaupt starten zu können. Außerdem können Funktionalitäten über mehrere Applikationen hinweg wiederverwendet werden. {% capture what %} ### Was #### Decorator Module werden über den Decorator `@NgModule` erzeugt. Er sieht beispielhaft folgendermaßen aus: ```typescript @NgModule({ declarations: [ // pipes, components/directives known in the whole module AppComponent, BookListComponent // is now known in the whole module ], imports: [ // depends on other modules BrowserModule // imports and re-exports most basic Angular directives ], providers: [], // list of services bootstrap: [AppComponent] // there is one root component }) export class AppModule {} // in most cases an empty class ``` Die Optionen dafür umfassen folgende Möglichkeiten: * `declarations`: Hier werden **Components / Directives und Pipes** angegeben. Diese gelten dann für das gesamte Modul. * `imports`: Hier werden **andere Module** angegeben. * `providers`: Hier werden **Services** angegeben. * `bootstrap`: Hier wird die **Root-Component** angegeben. Dies darf nur einmal innerhalb einer Applikation angegeben werden. (mehr im nächsten Abschnitt) {% endcapture %} #### bootstrapModule Neben der Deklaration den Root-Komponente muss für in einem Modul, muss dieses Modul einmal über die `bootstrapModule`-Funktion aufgerufen werden. Dies ist vergleich mit der `main`-Methode in Programmiersprachen wie z.B. Java. ```typescript // app.module.ts @NgModule({ declarations: [AppComponent], bootstrap: [AppComponent] }) // main.ts import { platformBrowserDynamic } from '@angular/platform-browser-dynamic'; import { AppModule } from './app/'; platformBrowserDynamic().bootstrapModule(AppModule); ``` Wer sich fragt, was `platformBrowserDynamic()` bedeutet - Hier können unterschiedliche umgebungen angegeben werden. `platformBrowser()` z.B. für Mobile-Apps.
