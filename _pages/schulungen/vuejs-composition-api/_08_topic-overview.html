Viele kennen und lieben das JaveScript Framework Vue.js. Doch mit den Jahren gewann Vue.js immer mehr Funktionen und wurde für immer größere Projekte genutzt. Doch die alte Vue.js API hatte Schwierigkeiten, mit den höheren Anforderungen mitzuhalten. Der Code wurde unübersichtlich. Aus diesem Grund wurde das API-Konzept überdacht. Eine neue und flexiblere API wurde geschaffen – die Composition API.

Statt den Code nur mit Optionen innerhalb eines großen JavaScript-Objektes zu organisieren, wie es bisher standardmäßig durch die Options-API gemacht wird, kann der Code nun nach Funktionalitäten gebündelt geschrieben werden. Dadurch können sich wiederholende Code-Fragmente aus einer Komponente herausgenommen werden und als wiederverwendbaren Code in anderen Komponenten eingesetzt werden. Der große Vorteil: Dadurch verbessert sich das Organisieren und Strukturieren von Vue.js Anwendungen.

Für Entwickler heißt das: mehr Flexibilität, bessere Wartung und eine große Zeitersparnis.

Wer glaubt, dass jetzt alle Vue Anwendungen neu geschrieben werden müssen, um das Composition API zu nutzen, irrt. Denn die alte Object API wird weiterhin Bestandteil der neuen Composition API sein.
